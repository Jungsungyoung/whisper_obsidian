# 라이브 로그 패널 설계

## 목표
업로드 후 처리 진행 상황을 터미널 스타일의 로그 패널로 실시간 표시

## 방식
폴링(1.5초) + 백엔드 로그 누적 (기존 구조 유지)

## 백엔드 변경 (`main.py`)
- `job_status`에 `logs: list[str]` 추가
- `update()` 및 `on_transcribe_progress()` 에서 `[MM:SS] 메시지` 형식으로 로그 항목 append
- `/status/{job_id}` 응답에 `logs` 포함

## 프론트엔드 변경 (`static/index.html`)
- 기존 진행 UI 아래 로그 패널 추가 (고정 높이 200px, 스크롤)
- `lastLogIndex` 변수로 마지막으로 받은 로그 인덱스 추적
- 폴링마다 새 항목만 append, 자동 스크롤

## 로그 형식
```
[00:00] 업로드 완료
[00:03] Whisper 전사 시작... (CUDA)
[00:17] 전사 완료, 단어 정렬 중...
[00:47] 화자 분리 중...
[01:28] 변환 중...
[01:29] Gemini AI 분석 중...
[01:31] Vault 저장 완료
```
